# Ch1. Computer System Overview

## 1. Operating System 이란?

- 어플리케이션 프로그램의 실행을 조절하는 프로그램.

- 컴퓨터 하드웨어와 어플리케이션 사이의 인터페이스 제공.

- Three objectives : Convenience, Efficiency, Ability to evolve.

  > 편리성, 효율성. OS가 없는 컴퓨터는 무쓸모.

- 컴퓨터에서 작동하는 가장 중요한 SW.


<br/>

## 2. Instruction Exectuion

- 명령어 실행은 프로세서가 명령어를 메모리로부터 fetch한 다음 각 명령어를 실행.

- 이 과정을 종료할 때 까지 반복한다.


<br/>


## 3. Interrupts

- cpu가 프로그램을 실행하고 있을 때, 입출력 하드웨어 등의 장치나 예외상황이 발생하여 처리가 필요할 경우 cpu에게 알려서 처리하는 기술.
- 프로세서의 활용성을 향상시키기 위해 제공된다.


<br/>

## 5. Principle of Locality

- 지역성의 원리, CPU가 참조하는 주소가 지역에 모여져 있다는 의미.
- 메모리 접근은 시간적, 공간적으로 지역성을 가진다. 시간적인 지역성은 한 번 읽었던 코드를 다시 읽을 확률이 높다는 것을 의미한다.
- 공간적인 지역성은 코드를 읽을 때 현재 코드의 주변에 있는 코드를 읽을 확률이 높다는 것을 의미한다.



<br/>

## 6. Cache Memory

- Cpu는 메인 메모리를 다녀와야함.
- 빈번히 사용되는걸 캐시에 올려두면 굳이 메인 메모리를 다녀올 필요 없이 캐시를 통해서 빠르게 접근할 수 있다.


<br/>


## 7. I/O Techniques

1. Programmed I/O

> I/O 연산이 있으면 그 명령을 수행하기 위해 해당 I/O 모듈에 명령을 내린다. 여기서 처리기는 I/O 연산이 완료될 때 까지 I/O 모듈의 상태를 주기적으로 점검해야 한다는 단점이 있다.

2. Interrupt Driven I/O

> 프로세서가 모듈에게 I/O 명령을 내리고 다른 작업을 수행하는 것.

3. Direct Memory Access(DMA)

> DMA 모듈에 입출력을 명령하고 프로세서는 다른작업을 한다. 이 I/O 연산을 DMA 모듈에 위임한다.
